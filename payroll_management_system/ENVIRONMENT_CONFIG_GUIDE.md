## 🎯 Overview

The codebase is now **environment-friendly** with the following features:

✅ **Single Source of Truth**: All base URLs are managed in one place  
✅ **Build Variant Switching**: Change environment by switching build variant  
✅ **Same Endpoints**: API endpoints remain identical across environments  
✅ **Automatic Configuration**: No manual code changes needed  
✅ **Validation**: Automatic URL validation on app startup

## 📁 Configuration Files

### 1. **build.gradle.kts** (app level)
Location: `/app/build.gradle.kts`

This file defines environment-specific base URLs:

```kotlin
buildTypes {
    debug {
        // Localhost/Development URLs
        buildConfigField("String", "PMS_BASE_URL", "\"http://10.0.2.2:8080/\"")
        buildConfigField("String", "EMS_BASE_URL", "\"http://10.0.2.2:8081/\"")
        buildConfigField("String", "ENVIRONMENT", "\"development\"")
    }
    release {
        // AWS/Production URLs
        buildConfigField("String", "PMS_BASE_URL", "\"https://dd6gzv507q8ms.cloudfront.net/\"")
        buildConfigField("String", "EMS_BASE_URL", "\"https://d3lpelprx5afbv.cloudfront.net/\"")
        buildConfigField("String", "ENVIRONMENT", "\"production\"")
    }
}
```

### 2. **ApiConfig.kt**
Location: `/app/src/main/java/com/example/myapplication/data/api/ApiConfig.kt`

Central configuration manager that:
- Reads URLs from BuildConfig
- Provides endpoint constants
- Validates configuration
- Logs environment details

### 3. **RetrofitClient.kt**
Location: `/app/src/main/java/com/example/myapplication/data/api/RetrofitClient.kt`

Uses ApiConfig to create API clients with proper base URLs.

## ⚙️ How It Works

### Architecture Flow

```
┌─────────────────────┐
│   build.gradle.kts  │  ← Define URLs per environment
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│    BuildConfig      │  ← Auto-generated by Gradle
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│    ApiConfig.kt     │  ← Central config manager
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│  RetrofitClient.kt  │  ← Creates API clients
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│   ViewModels        │  ← Use API clients
└─────────────────────┘
```

### Key Benefits

1. **No Hardcoded URLs in Code**
   - All URLs are in `build.gradle.kts`
   - Code reads from BuildConfig

2. **Endpoints Stay the Same**
   - `/api/auth/login` works on both localhost and AWS
   - Only base URL changes

3. **Automatic Validation**
   - App validates URLs on startup
   - Logs configuration for debugging

## 🔄 Switching Environments

### Method 1: Android Studio UI

1. Click **Build Variants** (bottom-left corner)
2. Select variant:
   - **debug** → Localhost/Development
   - **release** → AWS/Production
3. Rebuild and run

### Method 2: Command Line

```bash
# Debug (localhost)
./gradlew clean assembleDebug
./gradlew installDebug

# Release (AWS)
./gradlew clean assembleRelease
./gradlew installRelease
```

### Method 3: Gradle Tasks

```bash
# See all available variants
./gradlew tasks --all | grep assemble
```

## 🎨 Custom URLs

### Scenario 1: Different Localhost Ports

If your backend runs on different ports:

**Edit** `/app/build.gradle.kts`:
```kotlin
debug {
    buildConfigField("String", "PMS_BASE_URL", "\"http://10.0.2.2:3000/\"")
    buildConfigField("String", "EMS_BASE_URL", "\"http://10.0.2.2:3001/\"")
}
```

### Scenario 2: Staging Environment

Add a new build type:

**Edit** `/app/build.gradle.kts`:
```kotlin
buildTypes {
    debug {
        // ... existing config
    }
    
    create("staging") {
        initWith(buildTypes.getByName("debug"))
        buildConfigField("String", "PMS_BASE_URL", "\"https://staging-pms.example.com/\"")
        buildConfigField("String", "EMS_BASE_URL", "\"https://staging-ems.example.com/\"")
        buildConfigField("String", "ENVIRONMENT", "\"staging\"")
    }
    
    release {
        // ... existing config
    }
}
```

### Scenario 3: Different AWS URLs

**Edit** `/app/build.gradle.kts`:
```kotlin
release {
    buildConfigField("String", "PMS_BASE_URL", "\"https://your-new-pms-url.amazonaws.com/\"")
    buildConfigField("String", "EMS_BASE_URL", "\"https://your-new-ems-url.amazonaws.com/\"")
}
```

### Important Notes

⚠️ **Android Emulator Localhost**:
- Use `10.0.2.2` instead of `localhost` or `127.0.0.1`
- This is Android's special alias to host machine

⚠️ **URL Format**:
- Always end with `/` (e.g., `http://10.0.2.2:8080/`)
- Include protocol (`http://` or `https://`)

⚠️ **After Changing URLs**:
- Run `./gradlew clean` to regenerate BuildConfig
- Rebuild the project

## 🐛 Troubleshooting

### Issue 1: BuildConfig Not Found

**Error**: `Unresolved reference: BuildConfig`

**Solution**:
```bash
./gradlew clean build
```
Or in Android Studio: **Build** → **Clean Project** → **Rebuild Project**

### Issue 2: Wrong URLs Being Used

**Symptom**: App connects to wrong environment

**Check**:
1. Verify active build variant in Android Studio
2. Check Logcat for `ApiConfig` logs
3. Run: `./gradlew clean` and rebuild

### Issue 3: Connection Refused

**Symptom**: `Failed to connect to /10.0.2.2:8080`

**Solutions**:
1. Ensure backend server is running
2. Verify correct port number
3. Check firewall settings
4. For physical device, use your computer's IP instead of `10.0.2.2`

### Issue 4: SSL Errors on AWS

**Symptom**: SSL handshake failed

**Solutions**:
1. Ensure URL uses `https://`
2. Verify SSL certificate is valid
3. Check AWS CloudFront configuration

### Issue 5: Endpoints Not Working

**Symptom**: 404 Not Found errors

**Check**:
1. Verify backend is deployed and running
2. Check endpoint paths match backend routes
3. Review API documentation
4. Test with Postman/curl first

## 🔒 Security Notes

### For Production

1. **Never commit real production URLs to public repos**
2. Use environment variables or CI/CD secrets
3. Consider using `local.properties` for sensitive URLs

Example with `local.properties`:

```kotlin
// In build.gradle.kts
val localProperties = java.util.Properties()
val localPropertiesFile = rootProject.file("local.properties")
if (localPropertiesFile.exists()) {
    localProperties.load(localPropertiesFile.inputStream())
}

buildTypes {
    release {
        buildConfigField("String", "PMS_BASE_URL", 
            "\"" + localProperties.getProperty("PMS_BASE_URL", "https://default.com/") + "\"")
    }
}
```

Then add to `local.properties`:
```properties
PMS_BASE_URL=https://your-secret-pms-url.com/
EMS_BASE_URL=https://your-secret-ems-url.com/
```

## 📝 Best Practices

✅ **Do**:
- Use build variants for environment switching
- Keep endpoints consistent across environments
- Validate URLs on app startup
- Log configuration for debugging
- Test on both environments before release

❌ **Don't**:
- Hardcode URLs in source files
- Mix localhost and AWS URLs
- Commit sensitive production URLs to Git
- Forget to clean build after URL changes
- Use `localhost` in Android (use `10.0.2.2`)
